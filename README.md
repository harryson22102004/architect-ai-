> Turn a sentence into a production-ready architecture.
> **Powered by Google Gemini 2.5 Flash with native structured JSON output.**

## ✨ Why it wins

Most "AI" hackathon projects are thin conversational wrappers. ArchitectAI uses Gemini as a **deterministic structured-output engine** to perform real software-engineering tasks:

* It takes an idea like *"a clone of Airbnb with hosts, listings, bookings, reviews, and messaging"*
* And returns a **strictly typed JSON object** containing a complete Prisma schema, entity graph, and REST routing plan
* Then renders it as a side-by-side code + diagram dashboard

This proves Gemini can handle **multi-faceted technical reasoning** in a single, schema-constrained call — far beyond chat-completion.

## 🧩 The core technical trick

Inside [`lib/gemini.js`](lib/gemini.js) we declare a strict JSON schema and pass it to Gemini via `responseSchema`. This forces the model to return *only* parseable JSON matching our exact contract:

```js
const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });

await ai.models.generateContent({
  model: 'gemini-2.5-flash',
  contents: [{ role: 'user', parts: [{ text: SYSTEM_PROMPT + userPrompt }] }],
  config: {
    responseMimeType: 'application/json',
    responseSchema: architectureSchema, // ← prismaSchema, entities, edges, apiRoutes
    temperature: 0.7,
  },
});
```

The schema constrains Gemini to return:

| Field          | Type     | Used by                       |
| -------------- | -------- | ----------------------------- |
| `appName`      | string   | Header                        |
| `summary`      | string   | Subtitle                      |
| `prismaSchema` | string   | Syntax-highlighted code panel |
| `entities[]`   | object[] | React Flow nodes              |
| `edges[]`      | object[] | React Flow animated edges     |
| `apiRoutes[]`  | object[] | API tab with method badges    |

No regex parsing. No prompt-injection fragility. The shape is **guaranteed** by Gemini's structured-output mode.

## 🚀 Getting Started

### Prerequisites

* Node.js 20+
* MongoDB running locally (or a connection string)
* A **Gemini API key** → grab one at https://aistudio.google.com/apikey

### Setup

```bash
git clone <your-repo-url> architect-ai
cd architect-ai
yarn install
```

### Environment

Create a `.env` file in the project root:

```bash
MONGO_URL=mongodb://localhost:27017
DB_NAME=architectai
GEMINI_API_KEY=your_gemini_api_key_here
NEXT_PUBLIC_BASE_URL=http://localhost:3000
```

### Run

```bash
yarn dev
```

Open http://localhost:3000.

## 🔌 API Endpoints

| Method | Path                | Purpose                                                              |
| ------ | ------------------- | -------------------------------------------------------------------- |
| POST   | `/api/generate`     | Body: `{ prompt }`. Calls Gemini, persists, returns project object.  |
| GET    | `/api/projects`     | Returns latest 50 generated projects.                                |
| GET    | `/api/projects/:id` | Returns a single project.                                            |
| DELETE | `/api/projects/:id` | Removes a project from history.                                      |
